## Purpose

Context refresh ensures completed work updates CONTEXT so the ecosystem reflects the accepted state.

## Updated Content

Completed work may update:

- capabilities
- entities
- observations
- evidence
- events
- relationships
- decisions

## Refresh Rule

CONTEXT must reflect the validated state, not the proposed state.

Only validated and accepted outcomes are recorded as canonical ecosystem knowledge.

## Refresh Workflow

1. Validate the completed work.
2. Extract the accepted facts, decisions, and evidence.
3. Update the relevant CONTEXT records.
4. Link the context records to the Objective and supporting artefacts.
5. Confirm the updated state matches the validated outcome.

## Completion Rule

If the Context refresh is required, the Objective is not complete until the refresh is done.
