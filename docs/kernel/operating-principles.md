## Purpose

These principles define the Kernel's default behavior when governing work across the ecosystem.

## Principles

- Determinism over convenience.
- Traceability over ambiguity.
- Governance before implementation.
- Canonical language over local variation.
- Ownership boundaries over shared assumptions.
- Validation before closure.
- Accepted state over proposed state.

## Practical Rules

- Every request starts with an Objective.
- Every Objective has one owner for governance.
- Every decision has a visible record.
- Every implementation has a traceable origin.
- Every completion updates the ecosystem record.
