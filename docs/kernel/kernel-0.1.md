## Purpose

Kernel 0.1 defines the canonical operating model for the HOST ecosystem.

It turns the taxonomy from OBJ-001 into an operating system for governance, planning, delivery, validation, and context refresh.

## Scope

Kernel 0.1 covers:

- request lifecycle
- governance lifecycle
- Objective allocation
- decision workflow
- repository orchestration
- validation workflow
- AI operating rules
- traceability enforcement
- context refresh workflow

## Non-Scope

Kernel 0.1 does not include:

- application features
- product-specific business logic
- runtime automation
- workflow engines
- data processing code

## Core Rule

Every request must enter the ecosystem through the Kernel.

That means the request must be:

- identified
- classified
- allocated
- reviewed
- validated
- refreshed into CONTEXT

Only after those steps may implementation be approved.
