## Purpose

Validation confirms that completed work satisfies governance, ownership, and traceability requirements before the Objective is closed.

## Validation Requirements

Validation confirms:

- taxonomy compliance
- repository ownership respected
- numbering standards followed
- documentation updated
- CONTEXT updated
- Roadmap updated
- traceability complete

## Validation Inputs

- Objective ID
- request record
- decision record
- ADR if applicable
- implementation output
- supporting evidence
- affected repository acknowledgements

## Validation Outputs

- pass or fail outcome
- remediation notes if needed
- completion approval or rejection
- context refresh trigger

## Closure Rule

No Objective may be closed without validation.

If validation fails, the Objective remains open until the gap is resolved and revalidated.
