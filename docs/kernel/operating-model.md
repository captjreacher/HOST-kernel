## Purpose

The HOST Kernel operating model defines how governance moves from a request to a completed, traceable outcome.

It is the canonical process used by HOST, CONTEXT, Roadmap, and product repositories.

## Responsibilities

The Kernel owns:

- ecosystem governance
- Objective allocation
- lifecycle orchestration
- repository coordination
- traceability enforcement
- validation governance
- architecture compliance
- work admission
- work completion

The Kernel does not own:

- implementation
- product delivery detail
- repository-specific business logic
- data ownership inside CONTEXT
- planning detail inside Roadmap

## Operating Rules

- One request has one governing Objective.
- One Objective has one traceable path.
- One repository owns one responsibility boundary.
- One completed Objective must update the canonical records.
- No implementation begins before governance is complete.

## Canonical Artefacts

The operating model is expressed through:

- request lifecycle
- objective allocation
- decision framework
- governance workflow
- validation framework
- context refresh
- operating principles
- AI operating rules
