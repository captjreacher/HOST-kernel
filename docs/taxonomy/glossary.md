## Canonical Terms

| Term | Definition |
| --- | --- |
| Artefact | Any durable record produced by the ecosystem, including documents, issues, branches, and runtime records. |
| Registry | The canonical source of truth for a defined object family. |
| Taxonomy | The structured classification system used to organize ecosystem objects. |
| Ontology | The meaning model that defines how objects relate to each other. |
| Ownership | The repository or domain responsible for a given responsibility boundary. |
| Traceability | The ability to follow a record from origin to outcome and back again. |
| Lifecycle | The state progression an object follows from creation to retirement. |
| Canonical | The single approved definition used across the ecosystem. |
| Objective ID | The identifier that anchors all downstream artefacts for a unit of work. |
| Context Update | A durable knowledge record capturing what changed, what was learned, or what was observed. |

## Preferred Terms

- Use `Objective` instead of `goal` when referring to governed work intake.
- Use `Decision` instead of `choice` when the record matters for governance.
- Use `ADR` for formal architecture decisions.
- Use `Repository` for code and documentation boundaries.
- Use `Product` for externally delivered domains.
- Use `Context Update` for knowledge capture that must persist beyond the current discussion.

## Non-Preferred Terms

The following terms should not be used as canonical names:

- bucket
- folder
- story
- ticket
- note
- conversation log

These can appear informally, but they do not replace the registry terms.
